# ChArUco 캘리브레이션 보드 출력 & 부착 가이드

**목적**: 3대의 웹캠(LifeCam Studio / Samsung PC-CAM / Logitech StreamCam)을 ChArUco 보드 한 장으로 내부·외부 파라미터 모두 캘리브레이션하기.

---

## 1. 보드 규격 (기본값)

| 항목 | 값 |
|------|-----|
| 가로 칸(squares_x) | 6 |
| 세로 칸(squares_y) | 8 |
| 칸 크기(square) | 30 mm |
| ArUco 마커 크기 | 22 mm (칸의 약 73%) |
| 사전(dictionary) | DICT_5X5_100 |
| 보드 실측 크기 | **180 × 240 mm** |
| 용지 | A4 portrait (210 × 297 mm) |
| DPI | 600 |

모서리 코너 수 = `(6-1) × (8-1) = 35`개. 최소 10개 코너만 보여도 캘리브레이션 동작하므로 부분 가림에도 강합니다.

---

## 2. 출력 방법

### 2.1 생성

```bash
cd C:\Users\FITWIN\Desktop\biomech-mocap
python scripts/generate_charuco_a4.py
```

출력: `resources/charuco_board.png`

### 2.2 인쇄 설정 (★ 중요)

**반드시** 다음 설정으로 인쇄하세요:

- **용지**: A4
- **방향**: 세로(portrait)
- **배율(scale)**: **100% (실제 크기 / Actual size)**
- **여백 맞춤(Fit to page) 해제** — 이것을 켜면 보드가 축소/확대되어 캘리브레이션 정확도가 무너집니다.
- **양면 인쇄 OFF**
- **회색조**: OK (단, 선명도를 위해 300 DPI 이상 프린터 권장)

Windows 사진 뷰어에서 인쇄 시:
> Microsoft Print to PDF 또는 프린터 선택 → "맞춤(Fit picture to frame)" **체크 해제** → 기본 옵션의 "Actual size" 선택.

### 2.3 출력 후 검증 (반드시)

1. **자(ruler) 또는 디지털 캘리퍼**로 임의의 칸 하나의 **한 변 길이**를 잽니다.
2. 정확히 **30.0 mm**여야 합니다. ±0.5 mm까지 허용, 그 이상 차이나면 재인쇄하세요.
3. 전체 보드 폭도 확인: **180 × 240 mm** 이내이어야 합니다.
4. 인쇄된 보드 하단에 표시된 정보(`square=30.0mm, board=180x240mm`)와 일치하는지 확인.

**스케일 오차가 1%이면 3D 재구성 오차가 그대로 1%로 전파됩니다.** 측정은 선택이 아닌 필수.

## 3. 보드 제작 (부착)

종이 그대로 사용하면 휘어져서 캘리브레이션 오차의 주원인이 됩니다. **평평한 판**에 반드시 부착하세요.

### 3.1 추천 재료 (우선순위 순)

| 재료 | 두께 | 평탄도 | 비고 |
|------|------|--------|------|
| **알루미늄 복합판(디본드/Dibond)** | 3 mm | ◎ | 최고 선택, 가격↑ |
| **폼보드(Foam board)** | 5 mm | ○ | 저렴, 쉽게 구함 |
| **MDF 합판** | 3–5 mm | ○ | 무거움 |
| **두꺼운 하드보드지(2 mm)** | 2 mm | △ | 단기용 OK |
| 얇은 판지(1 mm) | 1 mm | ✗ | **피할 것** |

### 3.2 부착 절차

1. 출력한 A4 종이를 판 위에 놓고 **모서리 4곳을 정확히** 맞춥니다.
2. **스프레이 접착제(3M 77 등)** 또는 **양면테이프**로 전면 부착.
   - 액체 풀 금지(종이 울음 발생).
3. 롤러나 카드로 중앙→바깥으로 눌러 **기포 완전 제거**.
4. **무광 라미네이션** 필수 (선택이 아닌 필수).
   - 반사광이 있으면 ArUco 검출 실패.
   - 일반 A4 라미네이팅 파우치(80μm 이상) 사용.
   - 라미네이팅 전에 먼지 완전 제거.
5. 여분의 보드 **1개 더** 만들어 보관(분실/훼손 대비).

## 4. 부착(설치) 위치

### 4.1 내부 파라미터 캘리브레이션 (카메라별 개별)

카메라 한 대당 **15~25장** 촬영. 피험자 영역 = 포스 플레이트 중심에서 **1.5~3 m** 거리.

- 각 카메라 앞에서 보드를 **손으로 들고** 다음처럼 움직입니다:
  - FOV 4모서리 + 중앙 (위치 다양화)
  - 0°/15°/30°/45° 기울임 (각도 다양화)
  - 가까이(50 cm) ↔ 멀리(2 m) (거리 다양화)
  - 좌우 뒤집힘 포함

### 4.2 외부 파라미터 캘리브레이션 (3대 동시)

1. 보드를 포스 플레이트 **중앙 바닥**에 납작하게 둔 상태로 시작.
2. 이후 보드를 서서히 움직이며 3대 카메라가 **모두** 동시에 보는 위치에서 10초 이상 유지.
3. 완전히 동시에 볼 필요 없이 체인(예: Cam1+Cam2, Cam2+Cam3)으로도 가능.

### 4.3 세계 좌표계(world frame) 정의 — **이 단계가 핵심**

**보드를 포스 플레이트 위에 아래와 같이 놓습니다:**

```
        (포스 플레이트 위에서 내려다본 그림)

          Y+ (피험자 앞 방향)
           │
           │
  +─────────────────────────+
  │                         │
  │     Board1 (좌)│Board2(우)│
  │                         │
  │     ChArUco ┐           │
  │     ┌───────┘           │
  │     │  ★ (0,0) = 보드   │
  │     │  좌하단 모서리      │
  │     │  = 플레이트 좌하단  │
  │     └───────────────────┘
  └──┬──────────────────────┴──→ X+ (Board1→Board2 방향)
     원점
```

- ChArUco 보드의 **좌하단 코너(0,0)** 를 **포스 플레이트 좌하단 모서리(0,0 mm)** 에 정확히 맞춥니다.
- 보드 X축(가로, 6칸 방향)이 **플레이트의 +X(Board1→Board2)** 방향과 일치하도록 정렬.
- 이 상태로 3대 카메라에서 각각 1–2초 촬영 → `scripts/calibrate_world_frame.py` 가 카메라 ↔ 플레이트 좌표 변환을 계산.

## 5. 캘리브레이션 체크리스트

출력한 보드가 준비되면, 캘리브레이션 직전 아래 항목을 반드시 확인:

- [ ] 보드 칸 크기 측정값 = 30.0 mm ± 0.5 mm
- [ ] 보드 전체 크기 = 180 × 240 mm ± 1 mm
- [ ] 라미네이팅 완료, 반사광 없음
- [ ] 판이 완전히 평평 (눌렀을 때 휘지 않음)
- [ ] 조명 균일 (직접광·그림자 없음)
- [ ] 각 카메라의 **자동 노출/화이트밸런스 OFF**
- [ ] 카메라 초점 고정 (줌/포커스 잠금)

모든 항목 체크되면 `scripts/calibrate_intrinsics.py` (다음 단계) 실행.

## 6. 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| 재투영 오차 > 1.0 px | 보드 휨 / 종이만 사용 | 판 부착 재제작 |
| 마커 검출 실패 | 반사광 / 라미네이팅 누락 | 무광 라미네이팅 |
| 일부 각도에서만 검출 성공 | 이미지 수 부족 | 25장 이상 다양한 각도 재촬영 |
| 카메라 사이 스케일 불일치 | 프린트 배율 오차 | 출력 후 실측으로 `squareLength` 보정 |
| ArUco 인식률 저조 | 사전(dict)이 너무 큰 비트 | DICT_4X4_50 으로 변경 (더 작은 비트) |
